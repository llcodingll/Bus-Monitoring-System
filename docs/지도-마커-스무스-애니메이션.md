# 지도 마커 스무스 애니메이션

> 작업일: 2026-06-26
> 대상: `BusMap.vue` — Leaflet 마커 위치 갱신

## 해결한 문제

### 1. 7초 폴링 간격에서 마커 점프 제거

폴링 주기마다 마커를 새 좌표로 즉시 이동시키면 버스가 지도 위를 순간이동한다.
백엔드 tick 주기(7초)와 폴링 주기를 동일하게 맞추고, 애니메이션도 7초에 걸쳐 보간한다.
이렇게 하면 애니메이션이 끝나는 시점에 새 폴링 데이터가 정확히 도착해 마커 Lag 없이
실제 버스 이동 속도를 추종한다.

```ts
const ANIMATION_DURATION_MS = 7_000    // 백엔드 tick 주기(7s)와 동기화된 보간 시간
const ANIMATION_STEP_MS = 200          // 위치 갱신 주기 — 500ms에서 단축해 보간 부드러움 극대화

function smoothMove(busId: number, marker: L.Marker, targetLat: number, targetLng: number): void {
  const existing = animIntervals.get(busId)
  if (existing) clearInterval(existing)    // 이전 애니메이션 취소

  const start = marker.getLatLng()
  const startTime = Date.now()

  const id = setInterval(() => {
    const progress = Math.min((Date.now() - startTime) / ANIMATION_DURATION_MS, 1)
    marker.setLatLng([
      start.lat + (targetLat - start.lat) * progress,
      start.lng + (targetLng - start.lng) * progress,
    ])
    if (progress >= 1) clearInterval(id)
  }, ANIMATION_STEP_MS)

  animIntervals.set(busId, id)
}
```

폴링마다 새 좌표를 받으면 진행 중인 애니메이션을 중단하고 현재 위치에서 새 목표로 다시 시작한다.
progress는 `Math.min(..., 1)` 으로 clamp해 목표 좌표를 초과하지 않는다.

### 2. 마커 재생성 없이 아이콘·위치 갱신

폴링마다 마커를 `remove()` 후 재생성하면 Leaflet DOM 조작이 반복되어 깜빡임이 발생한다.
마커를 `Map<number, L.Marker>` 에 캐싱하고, 폴링 시 `setLatLng()` / `setIcon()` 만 호출해 DOM 노드를 재사용한다.

```ts
const markers = new Map<number, L.Marker>()

function getOrCreateMarker(bus: BusSummary): L.Marker {
  if (markers.has(bus.id)) return markers.get(bus.id)!  // 캐시 히트
  const marker = L.marker([bus.currentLatitude, bus.currentLongitude], {
    icon: buildIcon(bus, false),
  })
  marker.on('click', () => emit('select', bus.id))
  marker.addTo(map!)
  markers.set(bus.id, marker)
  return marker
}

// 폴링 갱신 시
function syncMarkers(buses: BusSummary[]): void {
  buses.forEach(bus => {
    const existing = markers.get(bus.id)
    if (existing) {
      smoothMove(bus.id, existing, bus.currentLatitude, bus.currentLongitude)
      existing.setIcon(buildIcon(bus, bus.id === props.selectedBusId))  // DOM 재사용
    } else {
      getOrCreateMarker(bus)
    }
  })
}
```

운행 종료된 버스(응답에서 사라진 버스)는 `activeIds`로 탐지해 마커와 애니메이션 타이머를 함께 제거한다.

### 3. 언마운트 시 애니메이션 타이머 전체 정리

컴포넌트 언마운트 시 활성 중인 모든 `setInterval` 을 정리하지 않으면 타이머가 백그라운드에서 계속 실행된다.

```ts
onUnmounted(() => {
  animIntervals.forEach(id => clearInterval(id))
  animIntervals.clear()
  clearPath()
  map?.remove()
  map = null
})
```

`map?.remove()` 로 Leaflet이 생성한 DOM과 이벤트 리스너도 함께 정리한다.
