# 버스 현재 상태 비정규화

> 작업일: 2026-06-26
> 대상: `buses` 테이블, `GET /api/buses`, `GET /api/buses/{id}`

## 해결한 문제

### 1. 조회마다 gps_locations 서브쿼리 제거

버스 목록 조회 시 각 버스의 현재 위치(위경도)를 실시간으로 가져오려면 gps_locations에서 최신 레코드를 버스마다 찾아야 한다.

**비정규화 전 (N+1):**
```sql
SELECT * FROM buses;
-- 버스마다 반복 실행
SELECT latitude, longitude FROM gps_locations
WHERE bus_id = ?
ORDER BY recorded_at DESC LIMIT 1;
```
버스 N대 → 쿼리 N+1번 발생.

**비정규화 후 (채택):**
```java
// buses 테이블에 직접 컬럼 저장
private BigDecimal currentLatitude;
private BigDecimal currentLongitude;
private Integer currentSpeed;
```
GPS 수신 시 `gps_locations INSERT`와 `buses UPDATE`를 같은 트랜잭션에서 처리해 일관성을 보장한다.
조회 시 JOIN 없이 `buses` 단일 테이블로 현재 위치 반환 가능.

```java
// BusSimulationScheduler — 두 쓰기를 하나의 @Transactional로 묶음
bus.updateLocation(latitude, longitude, speed, currentStop, nextStop, now);
gpsLocationRepository.save(GpsLocation.builder()...build());
```

### 2. 정류장 좌표 계산 쿼리 제거 (current_stop_id / next_stop_id FK)

버스 상세에서 현재 정류장·다음 정류장 이름을 표시할 때의 대안:

| 대안 | 문제점 |
|------|--------|
| GPS 좌표로 가장 가까운 정류장 실시간 계산 | stops 풀스캔, 오차 발생 |
| route_stops에서 현재 구간 계산 | 방향·순서 판단 로직 복잡 |
| **buses에 current_stop_id, next_stop_id FK 직접 저장 (채택)** | LEFT JOIN 2회로 단일 쿼리 해결 |

```java
// Bus Entity
@ManyToOne(fetch = FetchType.LAZY)
@JoinColumn(name = "current_stop_id")
private Stop currentStop;

@ManyToOne(fetch = FetchType.LAZY)
@JoinColumn(name = "next_stop_id")
private Stop nextStop;
```

```java
// BusRepository — 단일 쿼리로 정류장 이름까지 조회
@Query("SELECT b FROM Bus b LEFT JOIN FETCH b.currentStop LEFT JOIN FETCH b.nextStop")
List<Bus> findAllWithStops();
```

버스가 정류장에 도착할 때마다 시뮬레이터가 `bus.updateLocation()`에서 `currentStop`, `nextStop`을 함께 갱신한다.

### 3. ONLINE 상태 컬럼 미저장 (쓰기 비용 절감)

| 대안 | 문제점 |
|------|--------|
| `buses.status ENUM` 컬럼 저장 | 서울 7,000대 × 5분마다 UPDATE = 분당 1,400건 불필요한 쓰기, 배치 실패 시 상태 불일치 |
| **`last_communication_at` 기준 조회 시점 계산 (채택)** | 컬럼 추가 없음, 항상 정확, 추가 쿼리 없음 |

```java
// OnlineStatusPolicy — 조회 시점에 계산
public BusStatus resolve(Instant lastCommunicatedAt, Instant now) {
    if (lastCommunicatedAt == null) return BusStatus.OFFLINE;
    return !lastCommunicatedAt.isBefore(now.minus(THRESHOLD))
            ? BusStatus.ONLINE : BusStatus.OFFLINE;
}
```

buses 테이블에는 `last_communication_at` 타임스탬프만 저장하고, ONLINE/OFFLINE 판정은 읽기 경로에서 수행한다.
